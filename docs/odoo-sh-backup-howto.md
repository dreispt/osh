# Getting an Odoo.sh backup into Osh

The feature is split into two commands:

- `osh backup download <source>` — fetches or dumps a backup and stores it in the project cache (`.osh/backups/`).
- `osh restore [<dump>]` — restores a local backup file into the current branch's database and neutralizes it. With no argument it picks the newest cached backup.

This guide covers the two ways to get a backup out of an Odoo.sh project.

> **Important:** `osh` is not affiliated with Odoo S.A. or the odoo.sh service. The facts below are taken from the official Odoo and odoo.sh documentation as of the research date.

---

## 1. Manual download from the odoo.sh web UI

This is the simplest way to obtain a full backup (database dump + filestore).

### What you get

A `.zip` file that contains:

- `dump.sql` — PostgreSQL database dump in plain SQL format.
- `filestore/` — attachments and binary fields.
- `manifest.json` — Odoo version, PostgreSQL version, and module list.

Official docs state: “Each backup includes the database dump, the filestore (attachments and binary fields), logs, and sessions.” [^1]

### Steps

1. Sign in to <https://odoo.sh> and open your project.
2. Select the branch you want to back up. Only **Production** and **Staging** branches support backups; development branches do not. [^2]
3. Open the **Backups** tab on the branch page.
4. Choose a backup in the list and click the download icon. Select the full backup option (with filestore) for a complete restore.
5. Save the `.zip` file locally.

Alternative route from the Builds view: click **Download DB dump**. [^3]

### Retention

- Production builds are backed up every 24 hours.
- On the production server you can reach: 7 daily, 4 weekly, and 3 monthly backups.
- Manual backups are kept for 7 days (limit of 5 per day).
- Staging builds have no automatic backups, but you can create manual ones.
- Development builds have no backups at all. [^2]

### Use with `osh restore`

Move or copy the downloaded file into the project cache and restore it:

```bash
mkdir -p .osh/backups
cp /path/to/backup.zip .osh/backups/backup.zip
osh restore .osh/backups/backup.zip
```

Or pass any absolute path:

```bash
osh restore /path/to/backup.zip
```

`osh restore` will restore `dump.sql`, extract the filestore, and neutralize the database.

## 2. SSH access and automated download

SSH is the only built-in automation path for reaching odoo.sh backup files.

### Set up your SSH key

1. Generate or reuse an SSH key. odoo.sh supports `ssh-rsa` and `ssh-ed25519` keys. [^4]
2. In odoo.sh, click your GitHub user in the top-right corner and choose **Profile**.
3. Paste your public key under **Add a key manually** and click **Add**. You can also import the key from GitHub. [^3]

### SSH command pattern

The documented format is:

```bash
ssh <build_id>@<domain>
```

For example:

```bash
ssh 25004381@my-user-my-repository-staging-25004381.dev.odoo.com
```

The exact build ID and domain for your branch are shown on the **SSH** button/tab in the odoo.sh interface. [^3]

### What is available inside the container

- The container provides `psql` for direct database access.
- `pg_dump` is **not** available on odoo.sh containers; attempting to use it returns permission errors. [^5]
- Daily backups generated by the platform are stored under `/home/odoo/backup.daily`.
- The filestore is located under `/home/odoo/data/filestore/`. [^6]

### Copy the SQL dump to your machine

1. List the available daily backups:

   ```bash
   ssh <build_id>@<domain> ls /home/odoo/backup.daily
   ```

   The file names end in `_daily.sql.gz` and there is usually a matching `.json` file. [^7]

2. Copy the dump with `scp`:

   ```bash
   scp <build_id>@<domain>:/home/odoo/backup.daily/<backup_file.sql.gz> .
   ```

   Example:

   ```bash
   scp 25004381@my-user-my-repository-staging-25004381.dev.odoo.com:/home/odoo/backup.daily/2023-09-07_010937-my-project-prod-25004381_daily.sql.gz .
   ```

3. (Optional) Also copy the filestore. Because the daily directory contains only the SQL dump, the filestore must be fetched separately:

   ```bash
   # SSH into the container and compress the filestore
   ssh <build_id>@<domain>
   cd /home/odoo/data
   zip -r filestore.zip filestore
   # Then scp filestore.zip back
   ```

### Automated download with `osh backup download`

Instead of copying the file manually, you can use the backup plugin to fetch the latest daily dump into the project cache:

```bash
osh backup download odoosh://my-user-my-repository-staging-25004381
```

`osh` expands the shorthand to `my-user-my-repository-staging-25004381.dev.odoo.com` and uses the numeric suffix as the SSH user. The longer forms are still accepted:

```bash
osh backup download odoosh://my-user-my-repository-staging-25004381.dev.odoo.com
osh backup download odoosh://25004381@my-user-my-repository-staging-25004381.dev.odoo.com
```

To also download the filestore and create a full `.zip` backup, add `--filestore`:

```bash
osh backup download odoosh://my-user-my-repository-staging-25004381 --filestore
```

The database name is read from the daily backup filename, and `osh` streams the filestore from `/home/odoo/data/filestore/<db_name>` over SSH into a zip that `osh restore` can restore directly.

This stores the `.sql.gz` file in `.osh/backups/`. To restore the newest cached backup:

```bash
osh restore
```

To see all cached backups and pick a specific one:

```bash
osh backup list
osh restore cache:1
```

`osh restore` will create the target database, restore the dump, and neutralize it. The filestore is not restored from a plain daily dump; use `--filestore` when downloading via `odoosh://` or download the full `.zip` via the web UI if you need attachments.

## Summary

| Method          | Command path                                                           | Full backup?                                          | Automation? | Authentication                        |
| --------------- | ---------------------------------------------------------------------- | ----------------------------------------------------- | ----------- | ------------------------------------- |
| Web UI download | copy `.zip` into `.osh/backups/`, then `osh restore`                   | Yes (zip with dump + filestore)                       | Manual only | odoo.sh/GitHub login                  |
| SSH daily dump  | `osh backup download odoosh://PROJECT-BRANCH-BUILD` then `osh restore` | Partial by default (add `--filestore` for a full zip) | Scriptable  | SSH key configured in odoo.sh profile |
